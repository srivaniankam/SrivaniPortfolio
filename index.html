Deep dive · Backend Microservice

Publisher Service

Motivity Labs Pvt Ltd · ACE Online · Software Engineer (Nov 2025 – Present)
~413 source files17 bounded contexts18-service platformSpring Boot 2.3 · Java 8

Engineered publisher-service, the Spring Boot microservice owning the content & commerce domain of ACE Online — course catalogues, subjects, store listings, coupons/referrals, scholarships, campaigns, and SEO. Designed around domain-driven bounded contexts with a mixed reactive/blocking MongoDB layer, event-driven SQS/SNS integration, and Feign/REST clients to peer services. Delivered a WAR-deployable, cloud-configured service with Redis caching, AWS S3 integration, and centralized exception handling.

Architecture

  • Owns catalogues, subjects, store listings, coupons, scholarships, campaigns, SEO & announcements across 17 bounded contexts (~413 source files).
  • Layered as controller → service → domain → repository, with a dedicated usecase orchestration layer for complex flows, keeping each feature area independently evolvable.

Integration & Messaging

  • Event-driven, asynchronous integration using AWS SQS FIFO consumers and SNS publishing (via an events-hub service) to sync subjects, catalogues, coupons & rewards — including failed-event persistence for reliability.
  • Peer & third-party integration through OpenFeign and RestTemplate clients — user, payments, sales, the event-publisher hub, and the MirrorFly chat provider — with a custom Feign error decoder mapping failures to a unified exception type.

Persistence & Infrastructure

  • Mixed reactive + blocking MongoDB persistence (Reactive Mono/Flux repositories alongside blocking ones), tuned per context for a servlet-stack runtime.
  • Redis caching, AWS S3 storage, and scheduled jobs; all configuration externalized through Spring Cloud Config for environment-specific deployments.

Cross-cutting & Quality

  • Standardized concerns: a centralized @ControllerAdvice exception handler, a custom @DTO binding/validation mechanism (custom RequestResponseBodyMethodProcessor), reusable pagination/error response shapes, and custom date (de)serializers.
  • Observability via Sentry, Elastic APM, Glowroot and Spring Cloud Sleuth distributed tracing; packaged as a deployable WAR with Dockerized builds.
  • Documented the architecture & cross-service contracts as a living knowledge layer (ARCHITECTURE.md + estate-wide ESTATE.md) to speed onboarding and safe cross-service changes.
